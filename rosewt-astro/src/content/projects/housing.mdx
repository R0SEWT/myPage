## Sistema

Pipeline de datos a escala para modelado predictivo de precios inmobiliarios en Dinamarca. Procesamiento de 1.5M registros de transacciones residenciales con 30+ años de datos históricos.

## Arquitectura

**Ingesta** de datos heterogéneos → **feature engineering** (geográfico, temporal, tipológico) → **benchmarking** de modelos → **interpretabilidad** para validación.

## Decisiones técnicas

- **Feature engineering** abordando encoding geográfico, estacionalidad temporal e interacciones entre tipos de propiedad
- **Benchmarking** XGBoost + Optuna vs. H2O AutoML: análisis de tradeoffs rendimiento-interpretabilidad entre familias de modelos
- **SHAP + LIME** para validar decisiones del modelo contra dinámicas conocidas del mercado y detectar data leakage

## Restricciones

- Datos con valores faltantes, tipos heterogéneos y drift temporal a lo largo de 30+ años
- Necesidad de interpretabilidad para validar que el modelo captura señales reales, no artefactos

**Stack:** H2O.ai, XGBoost, Optuna, SHAP, LIME, Pandas, Python
