## Sistema

Pipeline end-to-end de document intelligence para investigación agrícola en CIP (CGIAR). Procesa documentos de investigación en múltiples idiomas y formatos, desde ingesta hasta retrieval semántico para question answering.

## Arquitectura

Pipeline multi-etapa: **ingesta** (PDF/HTML) → **parsing** format-aware → **chunking semántico** → **embedding generation** → **vector indexing** (Qdrant) con metadata en MongoDB.

## Decisiones técnicas

- **Chunking semántico** sobre fixed-size: mejor coherencia en retrieval para documentos con estructura variable
- **Qdrant + MongoDB** para retrieval híbrido: dense vector search combinado con metadata filtering
- **Batched embeddings** con retry logic y rate-limit backoff para operar dentro de quotas de API
- **Metadata enrichment** vía LLM para mejorar precisión de retrieval y habilitar búsqueda facetada

## Restricciones

- Documentos con artefactos de OCR, formatting inconsistente y contenido multilingüe
- Rate limits de API que requieren batching, backoff y failure recovery
- Volúmenes grandes de documentos con estructuras heterogéneas

**Stack:** FastAPI, Qdrant, MongoDB, OpenAI Embeddings, Python
