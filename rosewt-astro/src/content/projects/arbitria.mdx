## Sistema

RAG system domain-specific para documentos legales de arbitraje en Perú. Permite question answering contextual sobre textos legales complejos con ambigüedad inherente.

## Arquitectura

**Ingesta** → **indexación** dual (documento + chunk) → **retrieval semántico** → **generación contextual** con LlamaIndex. Backend FastAPI, storage en PostgreSQL, containerizado con Docker.

## Decisiones técnicas

- **Indexación dual** (documento y chunk): balance entre recall global y relevancia granular
- **Ingesta robusta** para PDFs legales con layouts multi-columna, tablas embebidas y headers inconsistentes
- **Interfaz conversacional** que mantiene contexto entre turnos y resuelve referencias ambiguas contra el corpus

## Restricciones

- Documentos legales con estructura altamente variable y terminología técnica
- Queries ambiguas que requieren resolución de contexto contra múltiples documentos
- Necesidad de respuestas precisas en dominio donde errores tienen consecuencias reales

**Stack:** LlamaIndex, FastAPI, OpenAI API, PostgreSQL, Docker
