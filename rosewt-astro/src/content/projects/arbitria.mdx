## Sistema

Sistema de retrieval legal para documentos de arbitraje peruano. El proyecto se mantiene como trabajo restringido, por lo que el sitio resume el enfoque sin enlazar un repositorio público.

## Arquitectura

**Ingesta** → **indexación dual** (documento + chunk) → **retrieval semántico** → **QA contextual**. Backend FastAPI, storage en PostgreSQL y despliegue con Docker.

## Decisiones técnicas

- **Indexación dual** (documento y chunk): balance entre recall global y relevancia granular
- **Ingesta robusta** para PDFs legales con layouts multi-columna, tablas embebidas y headers inconsistentes
- **Evaluación de chunking** para entender el trade-off entre precisión contextual y recuperación global

## Restricciones

- Documentos legales con estructura altamente variable y terminología técnica
- Queries ambiguas que requieren resolución de contexto contra múltiples documentos
- Necesidad de respuestas precisas en dominio donde errores tienen consecuencias reales

**Stack:** LlamaIndex, FastAPI, OpenAI API, PostgreSQL, Docker
